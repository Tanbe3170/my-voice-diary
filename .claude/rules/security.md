# セキュリティガイドライン

## コミット前の必須チェック
- [ ] ハードコードされた秘密情報がない（APIキー、パスワード、トークン）
- [ ] すべてのユーザー入力がバリデーション済み
- [ ] 認証・認可が正しく実装されている
- [ ] レート制限が全エンドポイントに適用されている
- [ ] エラーメッセージが機密情報を漏らさない

## 秘密情報管理
- ソースコードにシークレットを絶対にハードコードしない
- 必ず環境変数（process.env）を使用
- 起動時に必須シークレットの存在を確認
- 漏洩の可能性がある場合は即座にローテーション

## プロジェクト固有のセキュリティ
- JWT認証: HS256のみ、alg:none攻撃防止、timingSafeEqual使用
- HMAC画像トークン: 5分有効期限、SHA-256
- レート制限: Upstash Redis、fail-closed設計
- CORS: lib/cors.jsで許可オリジン管理
- Redis冪等性: SETNX/GETパターンで重複防止

## エージェントサポート
- **security-reviewer**エージェントを認証・入力処理・API変更時にプロアクティブに使用

## セキュリティ問題発見時
1. 即座に作業を停止
2. security-reviewerエージェントを使用
3. CRITICAL問題を修正してから続行
4. 漏洩したシークレットをローテーション
5. コードベース全体で類似問題を確認
